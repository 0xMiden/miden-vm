
## miden::core::math::k1_scalar
| Procedure | Description |
| ----------- | ------------- |
| push_n | Pushes the secp256k1 group order `n` onto the operand stack as 8 u32 LE limbs (n[0] on top,<br />n[7] deepest).<br /><br />Stack: [...]  ->  [n[0], n[1], ..., n[7], ...]<br /> |
| add | Modular addition: c = (a + b) mod n, with a, b < n.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /><br />n is close to 2^256 so the integer sum a + b can occupy 257 bits. We use `overflowing_add`<br />to recover both the carry and the wrapped sum, then reduce in two cases:<br />- carry = 1: the integer sum is at least 2^256, which is greater than n. The canonical<br />residue is the wrapped sum plus delta = 2^256 - n. The result fits in u256 (it is<br />bounded above by n).<br />- carry = 0: the wrapped sum equals the integer sum. Subtract n when the sum is at least<br />n, keep it otherwise.<br /> |
| sub | Modular subtraction: c = (a - b) mod n, with a, b < n.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /><br />When a >= b, the result of `a - b` is already canonical. When a < b, the wrapping subtraction<br />produces `2^256 + (a - b)`; adding n mod 2^256 yields `a - b + n`, which lies in (0, n).<br /> |
| neg | Modular negation: c = (-a) mod n, with a < n.<br /><br />Stack transition:<br />[a0..a7, ...]  ->  [c0..c7, ...]<br /><br />Returns 0 when a = 0; otherwise n - a (which is in (0, n)).<br /> |
| mul | Modular multiplication: c = (a * b) mod n.<br /><br />Inputs must be u32-valid u256 values, with at least one operand canonical (< n). Normal scalar<br />callers pass two canonical operands; `inv_unsafe` consumers pair a canonical operand with the<br />(modular-but-possibly-non-canonical) inverse.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /> |
| inv | Canonical multiplicative inverse: returns c = a^(-1) mod n in [0, n), with a in (0, n).<br /><br />Wraps `inv_unsafe` with a final `c < n` assert. Use this when the inverse is exposed to code<br />that depends on canonical bytes (hashing, equality, returned to caller). Hot paths that<br />immediately multiply the result by a canonical operand can use `inv_unsafe` instead.<br /><br />Stack transition:<br />[a0..a7, ...]  ->  [c0..c7, ...]<br /> |
| inv_unsafe | UNSAFE fast multiplicative inverse: returns some c congruent to a^(-1) mod n, with a in<br />(0, n). The returned u256 is u32-valid per limb but NOT guaranteed to satisfy c < n.<br /><br />Use only when the result is immediately multiplied by a canonical operand through<br />`k1_scalar::mul` or `modmul_k1_scalar`, which returns a canonical residue. For any other use,<br />call `inv`.<br /><br />Traps with "k1_scalar::inv_unsafe: hint not a multiplicative inverse" on a non-inverse<br />hint (including `a = 0`).<br /><br />Stack transition:<br />[a0..a7, ...]  ->  [c0..c7, ...]<br /> |
| verify_glv_split | Hint-and-verify GLV decomposition of a scalar `k`. Reads `k` from `mem[k_addr..k_addr+8]`<br />(8 u32 LE limbs), asks the host to split it into signed `(k_a, k_b)` such that<br />`k = k_a + k_b * lambda mod n` with both magnitudes strictly less than `2^128`, and writes<br />the witness to `mem[out_addr..out_addr+18]`:<br />out_addr[0..8]:    \|k_a\| (8 u32 LE limbs)<br />out_addr[8..16]:   \|k_b\| (8 u32 LE limbs)<br />out_addr[16]:      sign_a (0 or 1; 1 iff k_a < 0)<br />out_addr[17]:      sign_b (0 or 1; 1 iff k_b < 0)<br /><br />Asserts:<br />- magnitudes fit in 128 bits (limbs 4..7 are zero)<br />- sign bits are boolean<br />- `(canonical_a + canonical_b * lambda) mod n == k`, where<br />`canonical_x = sign_x ? n - \|k_x\| : \|k_x\|`<br /><br />Stack transition:<br />[out_addr, k_addr, ...]  ->  [...]<br /><br />Local memory layout (mem[0..56]):<br />0..8     : \|k_a\|<br />8..16    : \|k_b\|<br />16       : sign_a<br />17       : sign_b<br />20..28   : canonical_a (= sign_a ? n - \|k_a\| : \|k_a\|), used by the relation check<br />28..36   : canonical_b<br />40..48   : k cached from k_addr<br />48       : out_addr cached<br />49       : k_addr cached<br /> |
