
## miden::core::math::k1_scalar
| Procedure | Description |
| ----------- | ------------- |
| add | Modular addition: c = (a + b) mod n, with a, b < n.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /><br />n is close to 2^256 so the integer sum a + b can occupy 257 bits. We use `overflowing_add`<br />to recover both the carry and the wrapped sum, then reduce in two cases:<br />- carry = 1: the integer sum is at least 2^256, which is greater than n. The canonical<br />residue is the wrapped sum plus delta = 2^256 - n. The result fits in u256 (it is<br />bounded above by n).<br />- carry = 0: the wrapped sum equals the integer sum. Subtract n when the sum is at least<br />n, keep it otherwise.<br /> |
| sub | Modular subtraction: c = (a - b) mod n, with a, b < n.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /><br />When a >= b, the result of `a - b` is already canonical. When a < b, the wrapping subtraction<br />produces `2^256 + (a - b)`; adding n mod 2^256 yields `a - b + n`, which lies in (0, n).<br /> |
| neg | Modular negation: c = (-a) mod n, with a < n.<br /><br />Stack transition:<br />[a0..a7, ...]  ->  [c0..c7, ...]<br /><br />Returns 0 when a = 0; otherwise n - a (which is in (0, n)).<br /> |
| mul | Modular multiplication: c = (a * b) mod n, with a, b < n.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /> |
| inv | Multiplicative inverse: c = a^(-1) mod n, with a in (0, n).<br /><br />Hint-and-verify: emits `U256_INV_K1_SCALAR_EVENT` to ask the host for `inv = a^(-1) mod n`,<br />then checks the result with one `modmul_k1_scalar` call (asserting `a * inv == 1 mod n`).<br />The MASM proc traps with "k1_scalar::inv: hint not a multiplicative inverse" if the host<br />returns a non-inverse, including when called with `a = 0`.<br /><br />Stack transition:<br />[a0..a7, ...]  ->  [c0..c7, ...]<br /> |
| verify_glv_split | Hint-and-verify GLV decomposition of a scalar `k`. Reads `k` from `mem[k_addr..k_addr+8]`<br />(8 u32 LE limbs), asks the host to split it into signed `(k_a, k_b)` such that<br />`k ≡ k_a + k_b·λ (mod n)` with both magnitudes strictly less than `2^128`, and writes<br />the witness to `mem[out_addr..out_addr+18]`:<br />out_addr[0..8]:    \|k_a\| (8 u32 LE limbs)<br />out_addr[8..16]:   \|k_b\| (8 u32 LE limbs)<br />out_addr[16]:      sign_a (0 or 1; 1 iff k_a < 0)<br />out_addr[17]:      sign_b (0 or 1; 1 iff k_b < 0)<br /><br />Asserts:<br />- magnitudes fit in 128 bits (limbs 4..7 are zero)<br />- sign bits are boolean<br />- `(canonical_a + canonical_b·λ) mod n == k`, where `canonical_x = sign_x ? n - \|k_x\| : \|k_x\|`<br /><br />Stack transition:<br />[out_addr, k_addr, ...]  ->  [...]<br /><br />Local memory layout (mem[0..56]):<br />0..8     : \|k_a\|<br />8..16    : \|k_b\|<br />16       : sign_a<br />17       : sign_b<br />20..28   : canonical_a (= sign_a ? n - \|k_a\| : \|k_a\|), used by the relation check<br />28..36   : canonical_b<br />40..48   : k cached from k_addr<br />48       : out_addr cached<br />49       : k_addr cached<br /> |
