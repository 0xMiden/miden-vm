
## miden::core::math::f_k1
| Procedure | Description |
| ----------- | ------------- |
| push_p | Pushes the secp256k1 base prime `p = 2^256 - 2^32 - 977` onto the operand stack as 8 u32 LE<br />limbs (p[0] on top, p[7] deepest).<br /><br />Stack: [...]  ->  [p[0], p[1], ..., p[7], ...]<br /> |
| add | Modular addition: c = (a + b) mod p, with a, b < p.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /><br />Since p is close to 2^256, the integer sum a + b can occupy 257 bits. We use<br />`overflowing_add` to recover both the carry and the wrapped sum, then reduce in two cases:<br />- carry = 1: the integer sum is at least 2^256, which is greater than p. The canonical<br />residue is the wrapped sum plus delta = 2^256 - p = 2^32 + 977. The result fits in u256<br />(it is bounded above by p).<br />- carry = 0: the wrapped sum equals the integer sum. Subtract p when the sum is at least p,<br />keep it otherwise.<br /> |
| sub | Modular subtraction: c = (a - b) mod p, with a, b < p.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /><br />When a >= b, the result of `a - b` is already canonical. When a < b, the wrapping subtraction<br />produces `2^256 + (a - b)`; adding p mod 2^256 yields `a - b + p`, which lies in (0, p).<br /> |
| neg | Modular negation: c = (-a) mod p, with a < p.<br /><br />Stack transition:<br />[a0..a7, ...]  ->  [c0..c7, ...]<br /><br />Returns 0 when a = 0; otherwise p - a (which is in (0, p)).<br /> |
| mul | Modular multiplication: c = (a * b) mod p.<br /><br />Inputs must be u32-valid u256 values, with at least one operand canonical (< p). Normal field<br />callers pass two canonical operands; `inv_unsafe` consumers pair a canonical operand with the<br />(modular-but-possibly-non-canonical) inverse.<br /><br />Stack transition:<br />[b0..b7, a0..a7, ...]  ->  [c0..c7, ...]<br /> |
| inv | Canonical multiplicative inverse: returns c = a^(-1) mod p in [0, p), with a in (0, p).<br /><br />Wraps `inv_unsafe` with a final `c < p` assert. Use this when the inverse is exposed to code<br />that depends on canonical bytes (hashing, equality, returned to caller). Hot paths that<br />immediately multiply the result by a canonical operand can use `inv_unsafe` instead.<br /><br />Stack transition:<br />[a0..a7, ...]  ->  [c0..c7, ...]<br /> |
| inv_unsafe | UNSAFE fast multiplicative inverse: returns some c congruent to a^(-1) mod p, with a in<br />(0, p). The returned u256 is u32-valid per limb but NOT guaranteed to satisfy c < p.<br /><br />Use only when the result is immediately multiplied by a canonical operand through `f_k1::mul`<br />or `modmul_k1_base`, which returns a canonical residue. For any other use, call `inv`.<br /><br />Traps with "f_k1::inv_unsafe: hint not a multiplicative inverse" on a non-inverse hint<br />(including `a = 0`).<br /><br />Stack transition:<br />[a0..a7, ...]  ->  [c0..c7, ...]<br /> |
