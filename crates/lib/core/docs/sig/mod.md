STARK-based signature verifier (e2_105 variant, 0-fold / no FRI).<br /><br />Verifies a STARK proof that the signer knows a secret key `sk` such that<br />`RPO8_permute(sk) = pk`, bound to a message via Fiat-Shamir.<br /><br /># Protocol overview<br /><br />The protocol has 4 verifier rounds with no FRI (0-fold). Soundness comes<br />entirely from direct DEEP polynomial consistency checks at query points.<br /><br />```text<br />Phase 0:  Init transcript from INSTANCE_SEED + pk + msg<br />Round 1:  Absorb witness_commitment  -> grind(0)  -> sample lambda<br />Round 2:  Absorb quotient_commitment -> grind(0)  -> sample z, compute z^8<br />Round 3:  Absorb OOD evaluations     -> grind(10) -> sample alpha, beta<br />Round 4:  Load DEEP poly coeffs      -> grind(14) -> sample 53 query indices<br />Precompute DEEP constants, then verify all 53 DEEP queries<br />```<br /><br /># Transcript<br /><br />Poseidon2 width-12 sponge (rate=8, capacity=4), matching Miden VM's `hperm`.<br />Initialized from a hardcoded INSTANCE_SEED (encodes protocol parameters)<br />combined with per-signature public data (pk + msg).<br /><br /># Data flow<br /><br />Public inputs arrive on the operand stack: [pk, msg].<br />Proof data arrives on the advice stack in consumption order:<br /><br />```text<br />Advice stack (consumed top to bottom):<br />witness_commitment  (4 felts)   -- Phase 1: padw adv_loadw<br />ali_nonce            (1 felt)   -- Phase 1: adv_push.1<br />quotient_commitment (4 felts)   -- Phase 2: padw adv_loadw<br />ood_nonce            (1 felt)   -- Phase 2: adv_push.1<br />deep_alpha_nd       (2 felts)   -- Phase 3: adv_push.2<br />OOD evaluations    (96 felts)   -- Phase 3<br />prox_nonce           (1 felt)   -- Phase 3: adv_push.1<br />DEEP coefficients (256 felts)   -- Phase 4: 32x adv_pipe<br />query_nonce          (1 felt)   -- Phase 4: adv_push.1<br />```<br /><br />Merkle authentication paths come from the advice store (mtree_get) and<br />leaf data from the advice map (adv.push_mapval).<br />
