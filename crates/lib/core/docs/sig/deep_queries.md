DEEP polynomial consistency checks for the 0-fold signature verifier.<br /><br /># What this module does<br /><br />For each of 53 query indices, verifies that the committed trace polynomials<br />(witness + quotient) are consistent with their OOD evaluations via the DEEP<br />polynomial identity:<br /><br />DEEP(x) = (f_red(z) - f_red(x)) / (z - x) + beta * (f_red(gz) - f_red(x)) / (gz - x)<br /><br />where f_red is the Horner reduction of all committed columns using alpha.<br /><br /># Column layout<br /><br />The Horner reduction covers 24 extension-field columns (all committed polynomials):<br />- Witness:  8 base-field columns (from the RPO8 state trace, lifted to EF)<br />- Quotient: 16 extension-field columns (15 quotient chunks + 1 DEEP mask R)<br /><br /># Data sources<br /><br />- OOD Horner fixed terms: read from memory at SIG_HORNER_FIXED_PTR (set in Phase 3)<br />- Per-query leaf data: read from advice stack via adv.push_mapval after mtree_get<br />- DEEP polynomial coefficients: read from memory at SIG_DEEP_POLY_PTR<br />- Precomputed constants: -z, -gz in memory<br />
