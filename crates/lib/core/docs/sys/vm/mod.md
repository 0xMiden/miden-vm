
## miden::core::sys::vm::mod
| Procedure | Description |
| ----------- | ------------- |
| init_miden_air_shape_state | Loads, validates, and caches the Miden AIR shape.<br /><br />The shape is part of the VM-specific recursive verifier wrapper, not the<br />generic STARK verifier. Advice supplies the dynamic AIR heights in fixed instance order:<br /><br />[log_core, log_chiplets, log_blakeg_compression]<br /><br />The And8Lookup height is fixed verifier setup and is stored from `AND8_LOOKUP_LOG_HEIGHT`.<br /><br />Soundness contract:<br />- RELATION_DIGEST is a compile-time MASM constant; advice never supplies it.<br />- The preprocessed trace commitment is a compile-time MASM constant for the<br />canonical AND8 table; advice never supplies it.<br />- Dynamic heights are advice-supplied, but the Fiat-Shamir replay observes the full<br />four-AIR shape after public inputs in instance order. Wrong heights diverge from<br />the prover transcript.<br />- Shape checks reject malformed advice before transcript work starts.<br /><br />Writes:<br />AIR_SHAPE_PTR<br />CORE_TRACE_LENGTH_LOG_PTR<br />CHIPLETS_TRACE_LENGTH_LOG_PTR<br />BLAKEG_COMPRESSION_TRACE_LENGTH_LOG_PTR<br />AND8_LOOKUP_TRACE_LENGTH_LOG_PTR<br />TRACE_LENGTH_LOG_PTR<br />ORDER_TAG_PTR<br />RELATION_DIGEST_PTR<br />PREPROCESSED_TRACE_COM_PTR<br /><br />Stack-neutral.<br /> |
| verify_proof | Verifies a STARK proof attesting to the correct execution of a program in the Miden VM.<br /><br />Security parameters (num_queries, query_pow_bits, deep_pow_bits, folding_pow_bits) are<br />loaded from the advice stack, validated against the acceptable security policy, and<br />stored in memory for use by the generic verifier.<br />The dynamic Miden AIR heights are also loaded from advice, validated, and cached with the<br />fixed And8Lookup height before the generic verifier runs.<br /><br />- The public inputs are composed of the input and output stacks, of fixed size equal to 16, as<br />well as the program and the kernel procedures digests.<br />- The proof uses four AIRs: Core, Chiplets, BlakeGCompression, and And8Lookup. Main and auxiliary<br />leaves are concatenated in proof order after LMCS alignment.<br /><br />Inputs:  []<br />Outputs: []<br /> |
