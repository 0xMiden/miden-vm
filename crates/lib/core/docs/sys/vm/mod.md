
## miden::core::sys::vm::mod
| Procedure | Description |
| ----------- | ------------- |
| init_miden_air_shape_state | Loads, validates, and caches the Miden AIR shape from advice.<br /><br />The shape is part of the VM-specific recursive verifier wrapper, not the<br />generic STARK verifier. It is supplied in fixed instance order:<br /><br />[log_core, log_chiplets, log_poseidon2, reserved=0]<br /><br />Soundness contract:<br />- RELATION_DIGEST is a compile-time MASM constant; advice never supplies it.<br />- Heights are advice-supplied, but the Fiat-Shamir replay observes them after<br />public inputs in instance order. Wrong heights diverge from the prover transcript.<br />- Shape checks reject malformed advice before transcript work starts.<br /><br />Writes:<br />AIR_SHAPE_PTR<br />CORE_TRACE_LENGTH_LOG_PTR<br />CHIPLETS_TRACE_LENGTH_LOG_PTR<br />POSEIDON2_PERMUTATION_TRACE_LENGTH_LOG_PTR<br />TRACE_LENGTH_LOG_PTR<br />ORDER_TAG_PTR<br />RELATION_DIGEST_PTR<br /><br />Stack-neutral.<br /> |
| verify_proof | Verifies a STARK proof attesting to the correct execution of a program in the Miden VM.<br /><br />Security parameters (num_queries, query_pow_bits, deep_pow_bits, folding_pow_bits) are<br />loaded from the advice stack, validated against the acceptable security policy, and<br />stored in memory for use by the generic verifier.<br />The Miden AIR shape is also loaded from advice, validated, and cached<br />before the generic verifier runs.<br /><br />- The public inputs are composed of the input and output stacks, of fixed size equal to 16, as<br />well as the program and the kernel procedures digests.<br />- The proof uses three AIRs: Core, Chiplets, and Poseidon2Permutation. Main and auxiliary<br />leaves are concatenated in proof order after LMCS alignment.<br /><br />Inputs:  []<br />Outputs: []<br /> |
