
## miden::core::sys::vm::public_inputs
| Procedure | Description |
| ----------- | ------------- |
| process_public_inputs | Processes the public inputs.<br /><br />This involves:<br /><br />1. Loading from the advice stack the fixed-length public inputs and storing them in memory<br />starting from the address pointed to by `public_inputs_address_ptr`.<br />2. Loading kernel procedure digests from the advice stack into memory at<br />`variable_length_public_inputs_address_ptr`.<br />3. Absorbing the statement public inputs into the transcript, matching `MultiAir::observe`:<br />[ len(FLPI), num_VLPI_groups, len(kernel_felts), 0, FLPI..., kernel_felts... ].<br />Full 8-felt blocks are absorbed directly; only the trailing partial block is buffered.<br />4. Observing the Miden AIR shape in the transcript.<br />5. Loading auxiliary randomness from the advice stack and reducing the kernel digests.<br /><br />The fixed length public inputs are stored as extension field elements and copied to scratch from<br />the advice words already on the stack. Kernel digests are stored as padded base-field records<br />for reduction, and also copied to scratch in natural order for transcript absorption.<br /><br />POSTCONDITION: the transcript input buffer holds the unflushed tail of the statement and shape<br />observations. The generic verifier observes the main-trace commitment next, then flushes before<br />drawing auxiliary randomness.<br /><br />Input:<br />- Operand stack: [...]<br />- Advice stack: [fixed_len_PI..., num_kernel_proc_digests,<br />kernel_digest_elements..., beta0, beta1, alpha0, alpha1, ...]<br />Output: [...]<br /> |
