
## miden::core::sys::mod
| Procedure | Description |
| ----------- | ------------- |
| truncate_stack | Removes elements deep in the stack until the depth of the stack is exactly 16. The elements<br />are removed in such a way that the top 16 elements of the stack remain unchanged. If the stack<br />would otherwise contain more than 16 elements at the end of execution, then adding a call to this<br />function at the end will reduce the size of the public inputs that are shared with the verifier.<br /><br />Input: Stack with 16 or more elements.<br />Output: Stack with only the original top 16 elements.<br /><br />Cycles: 17 + 11 * overflow_words, where `overflow_words` is the number of words needed to drop.<br /> |
| drop_stack_top | Drop 16 values from the stack.<br /> |
| log_precompile_request | Folds a precompile commitment into the rolling transcript via `log_precompile` and removes<br />the helper words produced by the instruction.<br /><br />Computes the per-call statement word `STMNT = Poseidon2::merge(COMM, TAG)` from the supplied<br />commitment halves, seats it at stack[4..8] (the HPERM rate1 lanes), and lets the underlying<br />`log_precompile` opcode fold it into the transcript state. Pads stack[0..4] and stack[8..12]<br />with zero words so the opcode's writes there don't clobber data deeper in the stack.<br /><br />Input: `[COMM, TAG, ...]`<br />Output: Stack with `COMM` and `TAG` consumed and the three helper words produced by<br />`log_precompile` dropped.<br /> |
| register_expr | Registers an expression-bodied node `(tag, payload)` in the deferred-computation DAG and<br />returns its content-addressed digest on top of the operand stack.<br /><br />Thin wrapper around `adv.register_deferred`: invokes the sys event (which leaves the operand<br />stack unchanged and pushes NODE_DIGEST onto the advice stack), drops the 12 input felts, and<br />pulls NODE_DIGEST back onto the operand stack.<br /><br />Input:  `[PAYLOAD_LO, PAYLOAD_HI, TAG, ...]`<br />Output: `[NODE_DIGEST, ...]`<br /> |
| register_chunk | Registers a chunk-bodied node in the deferred-computation DAG and returns its<br />content-addressed digest on top of the operand stack.<br /><br />Thin wrapper around `adv.register_deferred_chunk`: invokes the sys event (which reads `8n`<br />felts from memory at `ptr`, leaves the operand stack unchanged, and pushes NODE_DIGEST onto<br />the advice stack), drops the 5 input felts, and pulls NODE_DIGEST back onto the operand stack.<br /><br />Input:  `[TAG, ptr, ...]`<br />Output: `[NODE_DIGEST, ...]`<br /> |
| evaluate | Evaluates a previously-registered node identified by its digest. The schema's `reduce`<br />pushes the canonical body (or nothing, for predicates) onto the advice stack; this helper<br />just consumes the input digest.<br /><br />Input:  `[NODE_DIGEST, ...]`<br />Output: `[...]`<br />Advice stack: depends on canonical (see `adv.evaluate_deferred` docs).<br /> |
| log_node_digest | Folds a node digest into the rolling precompile-transcript via the `log_precompile` opcode.<br />The digest must reference a node that will reduce to TRUE under the installed schema —<br />typically the digest of a chunk-bodied predicate (signatures) or of an Eq predicate node<br />(hash precompiles).<br /><br />Pads the operand stack so `log_precompile`'s implicit `[_, STMNT, _, ...]` shape sees<br />NODE_DIGEST at stack[4..8] (the HPERM rate1 lanes), then drops the three helper words the<br />opcode writes to stack[0..12].<br /><br />Input:  `[NODE_DIGEST, ...]`<br />Output: `[...]` (the rolling transcript root advances; node is folded in)<br /> |
