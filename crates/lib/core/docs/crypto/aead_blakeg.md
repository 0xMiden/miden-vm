BlakeG/u32-XOR AEAD helpers.<br /><br />The encryption path uses `aead_stream`, which calls the AEAD stream chip to<br />derive BlakeG-XOF blocks from `K_CTR` and a counter. The chip XORs the stream<br />with plaintext felts and writes expanded ciphertext limbs.<br />Authentication covers expanded ciphertext with empty associated data.<br /><br />Security contract:<br />- callers must never reuse `(key, nonce)`;<br />- counter blocks under a fixed `K_CTR` must be unique;<br />- expanded-ciphertext authentication is the canonical AEAD tag direction;<br />- this construction is not nonce-misuse-resistant.<br /><br />Test vectors are checked against `miden_crypto::hash::eidos::aead_ref`.<br />


## miden::core::crypto::aead_blakeg
| Procedure | Description |
| ----------- | ------------- |
| derive_ctr_key | Derives the CTR chaining value (`K_CTR`) from a key and nonce.<br /><br />`K_MAC` uses a separate proc with the `AEAD_MAC_DOMAIN` init CV. This proc is<br />hardcoded for `AEAD_CTR_DOMAIN`.<br /><br />The caller must not reuse `(key, nonce)`.<br /><br />Input:  [key(4), nonce(4), ...]<br />Output: [K_CTR(4), ...]<br /> |
| derive_mac_key | Derives the MAC key (`K_MAC`) from a key and nonce.<br /><br />Output word layout: `[r0, r1, s0, s1]`, where `(r0, r1)` is the<br />quadratic-extension evaluation point and `(s0, s1)` is the final mask.<br /><br />Input:  [key(4), nonce(4), ...]<br />Output: [K_MAC(4), ...]<br /> |
| auth_empty_ad_expanded_with_scratch | Authenticates exact-length expanded ciphertext with empty associated data.<br /><br />MAC input:<br />`nonce(4) \|\| ciphertext(ciphertext_len) \|\| [ad_len=0, ciphertext_len]`<br /><br />`scratch_ptr` is temporary workspace for the initial partial batch and for<br />the final tail plus length fields.<br /><br />Input:  [K_MAC(4), nonce(4), ct_ptr, ciphertext_len, scratch_ptr, ...]<br />Output: [tag0, tag1, ...]<br /><br />Preconditions: `scratch_ptr` points to at least 16 writable Felts and does<br />not overlap `ct_ptr..ct_ptr + ciphertext_len`.<br /> |
| auth_empty_ad_expanded | Authenticates expanded ciphertext with empty associated data.<br /><br />MAC input:<br />`nonce(4) \|\| ciphertext(8*num_blocks) \|\| [ad_len=0, ct_len=8*num_blocks]`<br /><br />Input:  [K_MAC(4), nonce(4), ct_ptr, num_blocks, ...]<br />Output: [tag0, tag1, ...]<br /><br />Preconditions: `ct_ptr` is word-aligned and points to `num_blocks * 8`<br />expanded ciphertext-limb Felts.<br /> |
| encrypt_blocks_stream | Encrypts `num_blocks * 8` plaintext felts using the `aead_stream` fast path.<br /><br />Input:  [K_CTR(4), src_ptr, dst_ptr, counter, num_blocks, ...]<br />Output: [K_CTR(4), src_ptr + 8*num_blocks, dst_ptr + 16*num_blocks,<br />counter + num_blocks, ...]<br /><br />Preconditions: `src_ptr` and `dst_ptr` are word-aligned and non-overlapping<br />for every encrypted block. Plaintext length must be an exact multiple of 8 felts.<br />`num_blocks = 0` is a valid no-op. Counter blocks under the same `K_CTR` must<br />not repeat.<br /> |
| encrypt_felts_expanded | Encrypts `num_felts` plaintext Felts into expanded ciphertext limbs.<br /><br />Full 8-Felt blocks use `encrypt_blocks_stream`; a non-empty tail is copied<br />into local scratch, encrypted as one padded stream block, and copied back as<br />exactly `2 * tail` ciphertext-limb Felts.<br /><br />Input:  [K_CTR(4), src_ptr, dst_ptr, counter, num_felts, ...]<br />Output: [K_CTR(4), src_ptr + num_felts, dst_ptr + 2*num_felts,<br />counter + ceil(num_felts / 8), ...]<br /><br />Preconditions: `src_ptr` and `dst_ptr` are non-overlapping. The aligned<br />prefix pointers must be word-aligned; the tail may have any length in 0..7.<br />Counter blocks under the same `K_CTR` must not repeat.<br /> |
| decrypt_empty_ad | Decrypts and authenticates expanded ciphertext using advice plaintext.<br /><br />The plaintext is nondeterministic advice. This procedure verifies it by<br />re-encrypting into `scratch_ptr`, comparing the regenerated ciphertext with<br />the input ciphertext, and checking the tag over the input ciphertext.<br /><br />Input:  [key(4), nonce(4), src_ptr, dst_ptr, num_felts, scratch_ptr, ...]<br />Output: [...]<br /><br />Where:<br />- `src_ptr` points to `2 * num_felts` ciphertext-limb Felts followed by<br />`tag(2)`.<br />- `dst_ptr` receives `num_felts` plaintext Felts from advice.<br />- Advice supplies plaintext Felts in memory order; the first `adv_push`<br />returns the first plaintext Felt.<br />- `scratch_ptr` must fit the regenerated ciphertext and MAC scratch stream.<br /> |
