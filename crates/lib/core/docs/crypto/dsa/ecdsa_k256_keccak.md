
## miden::core::crypto::dsa::ecdsa_k256_keccak
| Procedure | Description |
| ----------- | ------------- |
| verify | Verifies a secp256k1 ECDSA witness using the `miden-crypto::ecdsa_k256_keccak` message,<br />public-key commitment, and signature-scalar conventions, encoded in the native limb ABI below.<br />Acceptance intentionally differs from the Rust verifier: high-s witnesses are accepted.<br /><br />Inputs:<br />Operand stack: [PK_COMM, MSG, ...]<br />Advice stack:  [QX[8] \| QY[8] \| SIG_R[8] \| SIG_S[8] \| ...]<br />Outputs:<br />Operand stack: [...]<br />Advice stack:  [...]<br /><br />Where:<br />- `PK_COMM`: Poseidon2 hash commitment of `QX[8] \|\| QY[8]`, where coordinates are native<br />little-endian numeric u32 limbs, one limb per Felt.<br />- `MSG`: single word / 4 felts representing the 32-byte message to verify.<br />- `QX[8]`, `QY[8]`: native public-key coordinates as little-endian u32 limbs.<br />- `SIG_R[8]`, `SIG_S[8]`: native signature scalars as little-endian u32 limbs.<br /><br />Local memory layout (element addresses):<br />- loc[0 ..16] : native public key, QX[8] \|\| QY[8].<br />- loc[16..24] : Keccak256 message/digest scratch buffer.<br /><br />No result word is pushed. The procedure traps on malformed native limbs, invalid/off-curve public<br />keys, invalid signature scalars, or failed signature verification.<br /><br />Low-s is intentionally not enforced: this procedure proves that some untrusted advice satisfies<br />the ECDSA equation, and `(r, s)` and `(r, n - s)` are equivalent witnesses. An adaptation that<br />verifies a committed or canonical Ethereum signature must bind the exact signature encoding inside<br />the VM and enforce low-s on that bound value.<br /><br />Advice is untrusted. This procedure binds QX/QY to `PK_COMM`, and validates the public key,<br />reduced Keccak prehash, and signature scalars through precompile-backed deferred evaluation.<br /> |
