
## miden::core::crypto::dsa::ecdsa_k256_keccak
| Procedure | Description |
| ----------- | ------------- |
| verify | Verifies an secp256k1 ECDSA signature compatible with `miden-crypto::ecdsa_k256_keccak`.<br /><br />Given a public key commitment and the original message, reconstructs the calldata expected by<br />the EcdsaK256KeccakPrecompile precompile's verify chunk (pk bytes, Keccak256(message), signature)<br />into a single word-aligned 40-felt local-memory buffer, then dispatches to `verify_prehash`.<br /><br />Inputs:<br />Operand stack: [PK_COMM, MSG, ...]<br />Advice stack:  [PK[9] \| SIG[17] \| ...]<br />Outputs:<br />Operand stack: []<br />Advice stack:  []<br /><br />Where:<br />- `PK_COMM`: Poseidon2 hash commitment of the 33-byte ECDSA public key<br />- `MSG`: single word (4 field elements) representing the message to verify<br />- `PK[9]`: 33-byte public key packed as 9 felts on advice stack<br />- `SIG[17]`: 65-byte signature packed as 17 felts on advice stack<br /><br />Local memory layout — the 40-felt buffer at locaddr.0 lays out as:<br />locaddr[0..9]    pk (33 bytes packed as 9 felts)<br />locaddr[9..12]   zero pad (alignment to felt 12)<br />locaddr[12..20]  keccak256(message) digest (8 felts)<br />locaddr[20..37]  sig (65 bytes packed as 17 felts)<br />locaddr[37..40]  zero pad (alignment to felt 40)<br />locaddr[40..48]  msg bytes scratch (8 felts, transient — only used during keccak hash)<br /><br />Local memory is implicitly zero-initialised at procedure entry, so the two pad regions stay<br />zero without explicit stores.<br /><br />Traps if the public key does not hash to `PK_COMM`, or if the signature fails verification<br />inside the precompile.<br /> |
| verify_prehash | Verifies an ECDSA signature with pre-hashed message via the deferred-DAG framework.<br /><br />The caller provides a single pointer to a contiguous 40-felt buffer with the precompile's<br />word-aligned layout:<br /><br />bytes[0..33]    pk<br />bytes[33..48]   zero pad (15 bytes)<br />bytes[48..80]   digest (32 bytes)<br />bytes[80..145]  sig (65 bytes)<br />bytes[145..160] zero pad (15 bytes)<br /><br />Registers the buffer as the EcdsaK256KeccakPrecompile verify chunk node, evaluates the<br />predicate (which runs `PublicKey::verify_prehash` inside the precompile's reduce), and folds<br />the chunk's digest into the deferred-DAG transcript.<br /><br />Traps with `PrecompileError::AssertionFailed` if signature verification fails.<br /><br />Input:  `[buf_ptr, ...]`<br />Output: `[...]`<br /> |
