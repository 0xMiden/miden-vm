
## miden::core::crypto::dsa::ecdsa_k256_keccak
| Procedure | Description |
| ----------- | ------------- |
| verify | Verifies a secp256k1 ECDSA signature using the `miden-crypto::ecdsa_k256_keccak` message format.<br /><br />Inputs:<br />Operand stack: [PK_COMM, MSG, ...]<br />Advice stack:  [QX[8] \| QY[8] \| SIG_R[8] \| SIG_S[8] \| ...]<br />Outputs:<br />Operand stack: [...]<br />Advice stack:  [...]<br /><br />Where:<br />- `PK_COMM`: Poseidon2 hash commitment of the compressed SEC1 public key encoding<br />(`prefix \|\| x_be`) packed as 9 little-endian packed-u32 felts.<br />- `MSG`: single word / 4 felts representing the 32-byte message to verify.<br />- `QX[8]`, `QY[8]`: native public-key coordinates as little-endian u32 limbs.<br />- `SIG_R[8]`, `SIG_S[8]`: native signature scalars as little-endian u32 limbs.<br /><br />Local memory layout (element addresses):<br />- loc[0 ..16] : native public key, QX[8] \|\| QY[8]. Required by<br />`ecdsa_secp256k1::assert_verify_prehash(pubkey_ptr, digest_ptr, sig_ptr)`.<br />- loc[16..32] : native signature, SIG_R[8] \|\| SIG_S[8]. Required by the same precompile ABI.<br />- loc[32..40] : reused scratch:<br />1. PK[0..8] while binding `PK_COMM` via Poseidon2;<br />2. Keccak256(MSG) digest/prehash buffer passed as `digest_ptr`.<br /><br />The 40 local elements are the minimal contiguous memory ABI needed for the raw verifier call:<br />public key (16) + signature (16) + prehash output (8). The SEC1 commitment scratch is deliberately<br />overlapped with the later prehash buffer so it does not require additional locals.<br /><br />Advice is untrusted. The wrapper binds QX/QY to `PK_COMM`, and the native verifier binds and<br />validates the public key, reduced prehash, and signature scalars through precompile-backed<br />deferred evaluation.<br /> |
| verify_prehash | Verifies a secp256k1 ECDSA signature over a pre-hashed message.<br /><br />Input:  [pubkey_ptr, digest_ptr, sig_ptr, ...]<br />Output: [...]<br /><br />Memory:<br />- `pubkey_ptr[0..8]`: QX as little-endian u32 limbs.<br />- `pubkey_ptr[8..16]`: QY as little-endian u32 limbs.<br />- `digest_ptr[0..8]`: raw prehash z as little-endian u32 limbs.<br />- `sig_ptr[0..8]`: SIG_R as little-endian u32 limbs.<br />- `sig_ptr[8..16]`: SIG_S as little-endian u32 limbs.<br /><br />No result word is pushed. The procedure traps on malformed native limbs, invalid/off-curve public<br />keys, invalid signature scalars, or failed signature verification.<br /> |
