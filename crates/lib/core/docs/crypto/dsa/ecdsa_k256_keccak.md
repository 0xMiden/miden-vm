
## miden::core::crypto::dsa::ecdsa_k256_keccak
| Procedure | Description |
| ----------- | ------------- |
| verify | Verifies a secp256k1 ECDSA signature using the `miden-crypto::ecdsa_k256_keccak` message format.<br /><br />Inputs:<br />Operand stack: [PK_COMM, MSG, ...]<br />Advice stack:  [QX[8] \| QY[8] \| SIG_R[8] \| SIG_S[8] \| ...]<br />Outputs:<br />Operand stack: [...]<br />Advice stack:  [...]<br /><br />Where:<br />- `PK_COMM`: Poseidon2 hash commitment of `QX[8] \|\| QY[8]`, where coordinates are native<br />little-endian numeric u32 limbs, one limb per Felt.<br />- `MSG`: single word / 4 felts representing the 32-byte message to verify.<br />- `QX[8]`, `QY[8]`: native public-key coordinates as little-endian u32 limbs.<br />- `SIG_R[8]`, `SIG_S[8]`: native signature scalars as little-endian u32 limbs.<br /><br />Local memory layout (element addresses):<br />- loc[0 ..16] : native public key, QX[8] \|\| QY[8].<br />- loc[16..24] : Keccak256 message/digest scratch buffer.<br /><br />No result word is pushed. The procedure traps on malformed native limbs, invalid/off-curve public<br />keys, invalid signature scalars, or failed signature verification.<br /><br />Low-s note: the signature only needs to satisfy the ECDSA equation with canonical, nonzero scalar<br />values; low-s normalization (`SIG_S <= n/2`) is not checked. Thus both `(r, s)` and `(r, n - s)`<br />verify when otherwise valid. If the signature encoding is protocol-visible or must be unique,<br />enforce `0 < s <= n/2` before calling or canonicalize upstream.<br /><br />Advice is untrusted. This procedure binds QX/QY to `PK_COMM`, and validates the public key,<br />reduced Keccak prehash, and signature scalars through precompile-backed deferred evaluation.<br /> |
