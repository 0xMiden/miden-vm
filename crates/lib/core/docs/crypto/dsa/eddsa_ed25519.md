
## miden::core::crypto::dsa::eddsa_ed25519
| Procedure | Description |
| ----------- | ------------- |
| verify | Verifies an Ed25519 EdDSA signature compatible with `miden-crypto::eddsa_25519_sha512`.<br /><br />Given a public key commitment and the original message, reconstructs the calldata expected by<br />the EddsaEd25519Precompile schema's verify chunk (pk, SHA512(R \|\| PK \|\| MSG), signature) into<br />a single 40-felt local-memory buffer at locaddr.0, then dispatches to `verify_prehash`.<br /><br />Inputs:<br />Operand stack: [PK_COMM, MSG, ...]<br />Advice stack:  [PK[8] \| SIG[16] \| ...]<br />Outputs:<br />Operand stack: []<br />Advice stack:  []<br /><br />Where:<br />- `PK_COMM`: Poseidon2 hash commitment of the 32-byte Ed25519 public key<br />- `MSG`: single word (4 field elements) representing the message to verify<br />- `PK[8]`: 32-byte public key packed as 8 felts on advice stack<br />- `SIG[16]`: 64-byte signature packed as 16 felts on advice stack<br /><br />Local memory layout — the chunk buffer at locaddr.0..40 is naturally word-aligned because all<br />three component byte-sizes (32, 64, 64) are multiples of 4:<br />locaddr[0..8]    pk       (32 bytes packed as 8 felts)<br />locaddr[8..24]   k_digest (SHA-512 output, 16 felts)<br />locaddr[24..40]  sig      (64 bytes packed as 16 felts)<br />locaddr[40..48]  msg bytes scratch (transient)<br />locaddr[48..72]  sha512 input scratch (24 felts = R \|\| PK \|\| MSG)<br /><br />Traps if the public key does not hash to `PK_COMM`, or if signature verification fails<br />inside the schema kernel.<br /> |
| verify_prehash | Verifies an EdDSA (Ed25519) signature with a pre-computed nonce digest via the deferred-DAG<br />framework.<br /><br />The caller provides a single pointer to a contiguous 40-felt buffer with the schema layout:<br /><br />bytes[0..32]    pk<br />bytes[32..96]   k_digest = SHA-512(R \|\| A \|\| message)<br />bytes[96..160]  sig<br /><br />Registers the buffer as the EddsaEd25519Precompile verify chunk node, evaluates the predicate<br />(which runs `PublicKey::verify_with_unchecked_k` inside the precompile's reduce), and folds<br />the chunk's digest into the deferred-DAG transcript. Traps with<br />`PrecompileError::AssertionFailed` if verification fails.<br /><br />Input:  `[buf_ptr, ...]`<br />Output: `[...]`<br /> |
