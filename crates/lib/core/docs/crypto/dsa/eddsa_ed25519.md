
## miden::core::crypto::dsa::eddsa_ed25519
| Procedure | Description |
| ----------- | ------------- |
| verify | Verifies an Ed25519 EdDSA signature compatible with `miden-crypto::eddsa_25519_sha512`.<br /><br />This wrapper mirrors `miden-crypto::eddsa_25519_sha512`: given a public key commitment and the<br />original message, it reconstructs the calldata expected by the precompile (public key bytes,<br />message, signature). The public key and signature are supplied via the advice stack, and can be<br />obtained with the `eddsa_sign` function.<br /><br />Inputs:<br />Operand stack: [PK_COMM, MSG, ...]<br />Advice stack:  [PK[8] \| SIG[16] \| ...]<br />Outputs:<br />Operand stack: []<br />Advice stack:  []<br /><br />Where:<br />- `PK_COMM`: VM-hash commitment of the 32-byte Ed25519 public key<br />- `MSG`: message word serialized as eight u32 limbs before host verification<br />- `PK[8]`: 32-byte public key packed as 8 field elements on advice stack<br />- `SIG[16]`: 64-byte signature packed as 16 field elements on advice stack<br /><br />Local memory layout (element addresses):<br />- locaddr[0..8]:   public key (32 bytes packed as 8 felts)<br />- locaddr[8..16]:  message (8 u32 limbs from `MSG`)<br />- locaddr[16..32]: signature (64 bytes packed as 16 felts)<br /><br />The procedure traps if:<br />- The public key does not hash to `PK_COMM` (invalid commitment)<br />- The signature verification fails<br /> |
