
## miden::core::crypto::hashes::keccak256
| Procedure | Description |
| ----------- | ------------- |
| hash_bytes | Computes Keccak256 hash of data stored in memory.<br /><br />Input: [ptr, len_bytes, ...]<br />Output: [DIGEST_U32[8], ...]<br /><br />Where:<br />- ptr: word-aligned memory address containing INPUT_U32[len_u32] where len_u32=⌈len_bytes/4⌉<br />- len_bytes: number of bytes to hash<br />- INPUT_U32[len_u32] ~ INPUT_U8[len_bytes] with u32 packing (unused bytes in final u32 must be 0)<br />- DIGEST_U32[8] = [d_0, ..., d_7] = Keccak256(INPUT_U8[len_bytes])<br /><br />Local memory layout (element addresses):<br />loc[0..4]   DIGEST_LO  (computed by precompile, returned to caller)<br />loc[4..8]   DIGEST_HI  (computed by precompile, returned to caller)<br />loc[8..12]  PREIMAGE_DIGEST  (held until eq node construction)<br />loc[12..16] LEAF_DIGEST      (held until eq node construction)<br /><br />Flow:<br />1. Register the preimage chunk node (`Keccak256::preimage` tag at precompile_id) over the bytes<br />in memory at `ptr`. `register_chunk` derives PREIMAGE_DIGEST in-circuit (a linear hash of the<br />`8*n_chunks` felts), binding it to the caller's memory. `n_chunks = max(1, ceil(len_bytes/32))`<br />— empty input is one zero chunk (empty chunk bodies are banned).<br />2. `adv.evaluate_deferred` runs Keccak256::hash via the precompile host-side, interns the canonical<br />Digest leaf, and pushes its `LEAF_TAG \|\| DIGEST_LO \|\| DIGEST_HI` onto the advice stack<br />(TAG first). The advice digest is untrusted; it is bound by step 4.<br />3. Save DIGEST_LO/HI to locals (caller return) and register the Digest leaf in-circuit<br />(`register_expr` over the read payload), yielding LEAF_DIGEST.<br />4. Build & register an Eq predicate node `(PREIMAGE_DIGEST, LEAF_DIGEST)` and fold its digest<br />into the transcript via `log_node_digest`. The verifier's rehydrate recomputes<br />Keccak256(preimage) and rejects a forged digest, so the eq is logged, not eagerly evaluated.<br />5. Restore DIGEST_LO/HI from locals for the caller.<br /><br />In-circuit hashing per call: the node digests are derived in-circuit by the `register_*` helpers<br />(`n_chunks` permutations for the preimage chunk, one each for the digest leaf and the eq node),<br />plus the `log_precompile` fold. The Keccak256 preimage hash itself runs only inside the precompile's<br />reduce (host-side).<br /> |
| hash | Computes Keccak256 hash of a single 256-bit input.<br /><br />Input: [INPUT_U32[8], ...]<br />Output: [DIGEST_U32[8], ...]<br /><br />Where<br />- DIGEST_U32[8] = [d_0, ..., d_7] = Keccak256(INPUT_U8[32])<br />- INPUT_U32[8] = [i_0, ..., i_7] = [INPUT_LO, INPUT_HI] ~ INPUT_U8[32] with u32 packing<br /> |
| merge | Merges two 256-bit digests via Keccak256 hash.<br /><br />Input: [INPUT_L_U32[8], INPUT_R_U32[8], ...]<br />Output: [DIGEST_U32[8], ...]<br /><br />Where<br />- INPUT_L_U32[8] = [l_0, ..., l_7] = [INPUT_L_LO, INPUT_L_HI] ~ INPUT_L_U8[32]<br />- INPUT_R_U32[8] = [r_0, ..., r_7] = [INPUT_R_LO, INPUT_R_HI] ~ INPUT_R_U8[32]<br />- DIGEST_U32[8] = [d_0, ..., d_7] = Keccak256(INPUT_L_U8[32] \|\| INPUT_R_U8[32])<br /> |
