
## miden::core::crypto::hashes::eidos
| Procedure | Description |
| ----------- | ------------- |
| init_chaining_word | Constructs `Eidos::init_chaining_word(0, n)`.<br /><br />Input:  [n, ...]<br />Output: [CV, ...]<br /> |
| init_chaining_word_in_domain | Constructs `Eidos::init_chaining_word(domain, n)`.<br /><br />Input:  [n, domain, ...]<br />Output: [CV, ...]<br /> |
| compress | Performs one BlakeG compression.<br /><br />Input:  [BLOCK_LO, BLOCK_HI, CV, ...]<br />Output: [BLOCK_LO, BLOCK_HI, DIGEST, ...]<br /> |
| digest | Extracts the digest from a post-compression state.<br /><br />Input:  [BLOCK_LO, BLOCK_HI, DIGEST, ...]<br />Output: [DIGEST, ...]<br /> |
| init_no_padding | Prepares the top of the stack with the hasher initial state.<br /><br />This procedure does not handle padding, so the caller must<br />consume an amount of data which is a multiple of the rate (2 words).<br /><br />Input: []<br />Output: [R0, R1, C, ...]<br /><br />Where R0, R1, C are three zero words representing the initial VM hasher state<br />(R0 on top of stack).<br /><br />Cycles: 12<br /> |
| copy_digest | Copies the digest to the top of the stack.<br /><br />It is expected to have the hasher state at the top of the stack at the beginning of the procedure<br />execution.<br /><br />Input: [R0, R1, C, ...]<br />Output: [DIGEST, R0, R1, C, ...]<br /><br />Where:<br />- `R0` is the first rate word (positions 0-3, on top of stack).<br />- `R1` is the second rate word (positions 4-7).<br />- `C` is the capacity word / digest (positions 8-11).<br /> |
| absorb_double_words_from_memory | Hashes the memory `start_addr` to `end_addr` given a three-word VM hasher state.<br /><br />This requires that `end_addr = start_addr + 8n` where n = {0, 1, 2 ...}, otherwise the procedure<br />will enter an infinite loop.<br /><br />Input: [R0, R1, C, start_addr, end_addr, ...]<br />Output: [R0', R1', C', start_addr', end_addr, ...]<br />where `start_addr' = end_addr`.<br /><br />Where:<br />- `R0` is the first rate word / digest (positions 0-3, on top of stack).<br />- `R1` is the second rate word (positions 4-7).<br />- `C` is the capacity word (positions 8-11).<br /><br />Cycles: 4 + 3 * words, where `words` is `end_addr - start_addr`.<br /> |
| hash_double_words | Hashes an aligned range of double words.<br /><br />The range is committed using the same Eidos framing as `hash_words`: the input length is bound<br />into the initial chaining word. This helper is kept for callers whose data is naturally stored<br />as 8-Felt records.<br /><br />Input: [start_addr, end_addr, ...]<br />Output: [HASH, ...]<br /> |
| hash_words_with_domain | Hashes the word-aligned memory range `[start_addr, end_addr)` with a domain identifier.<br /><br />Requires `start_addr <= end_addr`, `end_addr` is not inclusive.<br />Requires `start_addr` and `end_addr` to be word-aligned.<br /><br />Input: [domain, start_addr, end_addr, ...]<br />Output: [H, ...]<br /><br />Cycles:<br />- even words: 54 cycles + 3 * words<br />- odd words: 66 cycles + 3 * words<br />where `words` is `(end_addr - start_addr) / 4`.<br /> |
| hash_words | Hashes the word-aligned memory range `[start_addr, end_addr)`.<br /><br />Equivalent to `hash_words_with_domain` with `domain = 0`.<br /><br />Requires `start_addr <= end_addr`, `end_addr` is not inclusive.<br />Requires `start_addr` and `end_addr` to be word-aligned.<br /><br />Input: [start_addr, end_addr, ...]<br />Output: [H, ...]<br /><br />Cycles:<br />- even words: 55 cycles + 3 * words<br />- odd words: 67 cycles + 3 * words<br />where `words` is `(end_addr - start_addr) / 4`.<br /> |
| prepare_hasher_state | Initializes the hasher state required for the `hash_elements_with_state` procedure.<br /><br />Inputs:  [ptr, num_elements, domain]<br />Outputs: [R0, R1, C, ptr, end_pairs_addr, num_elements%8]<br /><br />Where:<br />- ptr is the memory address of the first element to be hashed. This address must be<br />word-aligned - i.e., divisible by 4.<br />- num_elements is the number of elements to be hashed.<br />- R0, R1, C are three words representing the hasher state (R0 on top).<br />- end_pairs_addr is the memory address at which the pairs of words end.<br />- num_elements%8 is the number of elements that did not fit into word pairs and should be<br />hashed separately.<br /> |
| hash_elements_with_state | Computes hash of Felt values starting at the specified memory address using the provided hasher<br />state.<br /><br />This procedure divides the hashing process into two parts: hashing pairs of words using<br />`absorb_double_words_from_memory` procedure and hashing the remaining values with `compress`<br />procedure.<br /><br />Inputs:  [R0, R1, C, ptr, end_pairs_addr, num_elements%8]<br />Outputs: [HASH]<br /><br />Where:<br />- ptr is the memory address of the first element to be hashed. This address must be<br />word-aligned - i.e., divisible by 4.<br />- R0, R1, C are three words representing the hasher state (R0 on top).<br />- end_pairs_addr is the memory address at which the pairs of words end.<br />- num_elements%8 is the number of elements that did not fit into word pairs and should be<br />hashed separately.<br />- HASH is the resulting hash of the provided memory values.<br /> |
| hash_elements_with_domain | Computes hash of Felt values starting at the specified memory address.<br /><br />This procedure divides the hashing process into two parts: hashing pairs of words using<br />`absorb_double_words_from_memory` procedure and hashing the remaining values with `compress`<br />procedure.<br /><br />Inputs:  [ptr, num_elements, domain]<br />Outputs: [HASH]<br /><br />Where:<br />- ptr is the memory address of the first element to be hashed. This address must be<br />word-aligned - i.e., divisible by 4.<br />- num_elements is the number of elements to be hashed.<br />- domain is the domain identifier mixed into the initial chaining word.<br />- HASH is the resulting hash of the provided memory values.<br /><br />Cycles:<br />- If number of elements divides by 8: 52 cycles + 3 * words<br />- Else: 185 cycles + 3 * words<br />where `words` is the number of quads of input values.<br /> |
| pad_and_hash_elements | Hashes Felt values after padding the logical input length to a full 8-Felt block.<br /><br />This is not a hasher-internal padding rule. It changes the committed stream from<br />`elements[..num_elements]` to `elements[..padded_len]`, where<br />`padded_len = ceil(num_elements / 8) * 8`. The padding zeros are synthesized by this helper;<br />callers do not need to store them in memory.<br /><br />Inputs:  [ptr, num_elements]<br />Outputs: [HASH]<br /> |
| hash | Computes the VM-native hash of a single word (256-bit input).<br /><br />Inputs:  [A]<br />Outputs: [B]<br /><br />Where:<br />- A is the word to be hashed.<br />- B is the resulting hash.<br /><br />Cycles: 19<br /> |
| merge | Merges two words (256-bit digests) via the VM-native hash.<br /><br />Inputs:  [A, B]<br />Outputs: [C]<br /><br />Where:<br />- A and B are the words to be merged.<br />- C is the resulting hash.<br /><br />Cycles: 16<br /> |
| merge_in_domain | Merges two words (256-bit digests) via the VM-native hash with a domain identifier.<br /><br />Inputs:  [domain, A, B, ...]<br />Outputs: [C, ...]<br /><br />Where:<br />- A and B are the words to be merged (A corresponds to the first rate word).<br />- C is `hash_elements_with_domain(A \|\| B, domain)`.<br /><br />Cycles: 16<br /> |
