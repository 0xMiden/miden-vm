
## miden::core::crypto::hashes::sha512
| Procedure | Description |
| ----------- | ------------- |
| hash_bytes | Computes SHA512 hash of data stored in memory.<br /><br />Input:  [ptr, len_bytes, ...]<br />Output: [DIGEST_U32[16], ...]<br /><br />Where:<br />- ptr: word-aligned memory address containing INPUT_U32[len_u32] where len_u32=⌈len_bytes/4⌉<br />- len_bytes: number of bytes to hash<br />- DIGEST_U32[16] = SHA-512(INPUT_U8[len_bytes])<br /><br />Local memory layout (element addresses):<br />loc[0..16]   DIGEST  (16 felts, used as both schema-output buffer and caller-return)<br />loc[16..20]  PREIMAGE_DIGEST<br />loc[20..24]  LEAF_DIGEST<br /><br />Flow mirrors keccak256::hash_bytes — see that file for the high-level explanation. The only<br />structural difference is that the SHA-512 digest leaf is a `Chunks(2)` self-evaluating chunk<br />node (16 felts) rather than a `Value` leaf, so the leaf registration uses register_chunk<br />against the digest buffer materialized in local memory.<br /> |
