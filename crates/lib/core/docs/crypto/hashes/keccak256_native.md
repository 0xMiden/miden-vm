
## miden::core::crypto::hashes::keccak256_native
| Procedure | Description |
| ----------- | ------------- |
| keccak256_bytes | Computes Keccak-256 of a byte buffer stored as u32-packed little-endian<br />felts in memory. Input convention matches the precompile path<br />(`hash_bytes` in `keccak256.masm`):<br /><br />- `src_ptr` points to `ceil(len_bytes / 4)` u32 felts.<br />- Each u32 packs four little-endian bytes: u32[k] = b[4k] \| (b[4k+1] << 8) \| ...<br />- If `len_bytes` is not a multiple of 4, the trailing bytes of the final<br />u32 must be zero (the proc relies on this when ORing the domain byte in).<br /><br />Stack transition:<br />[src_ptr, len_bytes, ...]  ->  [d_0, d_1, ..., d_7, ...]<br /><br />Where `d_0..d_7` are 8 u32 felts forming the 32-byte digest in little-endian<br />lane order (`d_k = state[k]` after the final permutation): bytes 0..32 of the<br />digest correspond to (d_0 bytes 0..4) \|\| (d_1 bytes 0..4) \|\| ... \|\| (d_7<br />bytes 0..4), matching `Keccak256::hash` output.<br /><br />Local memory layout (96 felts):<br />0..50   : sponge state (50 felts).<br />50, 51  : unused (alignment padding before scratch).<br />52..86  : final-block scratch (34 felts; word-aligned at 52).<br />86..96  : runtime counters --<br />86 : src_ptr (current; advanced by 34 per absorbed block)<br />87 : remaining bytes (decremented by 136 per absorbed block)<br />88 : final-block u32 count (= ceil(remaining / 4))<br />89 : copy-loop counter i<br />90 : final-block q = remaining / 4<br />91 : final-block r = remaining mod 4<br />92..96 : unused<br /> |
