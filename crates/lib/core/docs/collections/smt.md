
## miden::core::collections::smt
| Procedure | Description |
| ----------- | ------------- |
| get | Returns the value located under the specified key in the Sparse Merkle Tree defined by the<br />specified root.<br /><br />If no values had been previously inserted under the specified key, an empty word (i.e.,<br />[ZERO; 4]) is returned.<br /><br />Inputs:<br />Operand stack: [K, R, ...]<br /><br />Outputs:<br />Operand stack: [V, R, ...]<br /><br />Fails if the tree with the specified root does not exist in the VM's advice provider.<br /><br />Cycles<br />Leaf empty: 67 cycles<br />Leaf single: 162 cycles<br />Leaf multiple, found at first key: 294 + 239 * pair_count<br />Leaf multiple, found at last key: 242 + 291 * pair_count<br />Leaf multiple, missing before first key: 224 + 291 * pair_count<br />Leaf multiple, missing after last key: 224 + 291 * pair_count<br />Leaf multiple, middle keys or insertion points: between the first-key and last-key costs<br /> |
| set | Inserts the specified value under the specified key in a Sparse Merkle Tree defined by the<br />specified root. If the insert is successful, the old value located under the specified key<br />is returned via the stack.<br /><br />If the VALUE is an empty word (i.e., [ZERO; 4]), the new state of the tree is guaranteed to<br />be equivalent to the state as if the updated value was never inserted.<br /><br />Inputs:<br />Operand stack: [V, K, R, ...]<br />Outputs:<br />Operand stack: [V_old, R_new, ...]<br /><br />Fails if the tree with the specified root does not exist in the VM's advice provider.<br /><br />Cycles<br />Leaf empty<br />no-op removal: 108 cycles<br />insertion: 182 cycles<br />Leaf single<br />removal: 271 cycles<br />replacement: 296 cycles<br />insertion (leaf becomes multiple): 610 cycles<br />Leaf multiple<br />removal at first key: 587 + 320 * pair_count<br />removal at last key: 442 + 450 * pair_count<br />replacement at first key: 561 + 262 * pair_count<br />replacement at last key: 373 + 450 * pair_count<br />insertion before first key: 954 + 310 * pair_count<br />insertion after last key: 747 + 498 * pair_count<br />no-op removal after last key: 311 + 441 * pair_count<br />middle keys or insertion points: between the first-key and last-key costs<br /> |
| peek | Emits the "miden::core::collections::smt::smt_peek" event, which pushes onto the advice stack<br />the value associated with the specified key in a Sparse Merkle Tree defined by the specified<br />root.<br /><br />If no value was previously associated with the specified key, [ZERO; 4] is pushed onto the advice<br />stack.<br /><br />This is a fast, untrusted lookup. It reads from the advice provider and does not verify a Merkle<br />path from the supplied root to the returned leaf node. It also does not prove that the returned<br />value is the value committed under the supplied root.<br /><br />Caller code must verify the value before relying on it. The usual pattern is to call `peek`, then<br />later call `set`, and compare the old value returned by `set` with the value returned by `peek`.<br />That pattern lets a program avoid doing the same Merkle path verification twice.<br /><br />Inputs:<br />Operand stack: [KEY, ROOT]<br />Advice stack:  []<br /><br />Outputs:<br />Operand stack: [KEY, ROOT]<br />Advice stack:  [VALUE]<br /><br />Panics if:<br />- the tree with the specified root does not exist in the VM's advice provider.<br /><br />Cycles: 3 cycles<br /> |
