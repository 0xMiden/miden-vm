
## miden::core::stark::public_inputs
| Procedure | Description |
| ----------- | ------------- |
| process_public_inputs | Processes the public inputs.<br /><br />This generic procedure dispatches to the relation-specific public-input handler via `dynexec`.<br />The relation handler owns its public-input schema and Fiat-Shamir observation schedule.<br /><br />Inputs:  [...]<br />Outputs: [...]<br /><br />Invocation: exec<br /> |
| compute_and_store_public_inputs_address | Computes and stores the start address of the ACE public-input region.<br /><br />The public-input region stores `num_fixed_len_pi` base-field public values as extension-field<br />slots, i.e. two base-field memory cells per public value. It is placed immediately before the<br />aux-randomness word in ACE READ.<br /><br />This helper only computes the ACE READ public-input pointer. Relation-specific code owns any<br />additional statement inputs that are not read by the ACE circuit.<br /><br />Inputs:  [num_fixed_len_pi, ...]<br />Outputs: [...]<br /><br />Invocation: exec<br /> |
| load_base_store_extension_double_word_from_mem | Loads 8 base-field elements from memory, stores them as 8 extension-field elements, and leaves<br />the base elements in the rate slots for a direct full-rate absorption.<br /><br />Input:  [R1, R2, C, dst, src, ...]<br />Output: [R1', R2', C, dst + 16, src + 8, ...]<br /><br />Invocation: exec<br /> |
